The user wants me to explain the state features empirical expectation computation. This is very similar to the transition features part, with two key differences.
## 计算状态特征的经验期望 $E_{\tilde{P}}[s_l]$

这段代码和上面计算 `d3`（转移特征期望）的**结构几乎完全一样**，只有 **2 个区别**：

---

### 区别对比

| | `d3`（转移特征） | `d4`（状态特征） |
|---|---|---|
| 遍历特征 | `for k in range(n_transfer_features)` | `for l in range(n_state_features)` |
| 特征函数签名 | `t_k(y[i-1], y[i], x, i)` **4 个参数** | `s_l(y[i], x, i)` **3 个参数** |
| 位置范围 | `range(1, n_position)` **从 1 开始** | `range(n_position)` **从 0 开始** |

---

### 区别 1：特征函数参数不同

```python
# 转移特征 — 需要前一个状态 y[i-1] 和当前状态 y[i]
transfer_features[k](y[i-1], y[i], x, i)
#                     ~~~~~~   ~~~~
#                     起点      终点

# 状态特征 — 只需要当前状态 y[i]
state_features[l](y[i], x, i)
#                  ~~~~
#                  当前状态
```

这是由定义决定的：
- **转移特征**描述的是两个相邻状态之间的关系，所以需要 $y_{i-1}$ 和 $y_i$
- **状态特征**描述的是单个状态与观测值的关系，只需要 $y_i$

---

### 区别 2：位置范围不同

```python
# 转移特征：range(1, n_position) → 位置 1, 2（跳过位置 0）
for i in range(1, n_position):

# 状态特征：range(n_position) → 位置 0, 1, 2（全部位置）
for i in range(n_position):
```

原因：转移特征在位置 0 没有前一个状态 $y[-1]$，所以**必须从位置 1 开始**。而状态特征不依赖前一个状态，所以**每个位置都可以计算**。

---

### 数学公式

$$E_{\tilde{P}}[s_l] = \sum_{x, y} \tilde{P}(x, y) \cdot \sum_{i=0}^{n-1} s_l(y_i, x, i)$$

与 `d3` 的公式对比：

$$E_{\tilde{P}}[t_k] = \sum_{x, y} \tilde{P}(x, y) \cdot \sum_{i=1}^{n-1} t_k(y_{i-1}, y_i, x, i)$$

唯一的区别就是求和范围 $i$ 从 0 开始还是从 1 开始。

---

### 具体例子

以 $s_1$ 为例，假设 $y = (0, 1, 0)$, $x = (0, 1, 1)$：

```python
def s1(y0, x, i):
    return int(y0 in {0} and x in {(0,1,1), (1,1,0), (1,0,1)} and i in {0, 1, 2})
```

| 位置 $i$ | $y_i$ | $x$ | $i \in \{0,1,2\}$? | $x \in \{(0,1,1),...\}$? | 结果 |
|---|---|---|---|---|---|
| 0 | 0 | (0,1,1) | ✓ | ✓ | **1** |
| 1 | 1 | (0,1,1) | ✓ | ✓ | **0**（$y_i=1 \neq 0$）|
| 2 | 0 | (0,1,1) | ✓ | ✓ | **1** |

$s_1$ 在这个 $(x, y)$ 对上被激活 **2 次**。如果 $\tilde{P}(x, y) = 0.15$，则 $d4[0] += 0.15 \times 2 = 0.30$。

---

### `nn` 的累加是共享的

注意 `nn[xi][yi] += 1` 在 `d3` 和 `d4` 的计算中**都会执行**，所以 `nn` 统计的是**所有特征（转移 + 状态）被激活的总次数**：

```
nn[xi][yi] = ∑ₖ ∑ᵢ tₖ(y,y,x,i) + ∑ₗ ∑ᵢ sₗ(y,x,i)
```

这个 `nn` 后面在 IIS 迭代的 $e^{d \cdot f^\#(x,y)}$ 中会用到。
